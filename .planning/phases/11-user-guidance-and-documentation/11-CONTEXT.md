# Phase 11: User Guidance and Documentation - Context

**Gathered:** 2026-01-29
**Status:** Ready for planning

<domain>
## Phase Boundary

建立使用者指南頁面，幫助商家快速找到 PayUNi 功能位置、理解如何使用、解決常見問題。這是文件和導引功能，不包含新的支付功能。

</domain>

<decisions>
## Implementation Decisions

### 內容結構與導航
- **組織方式**：混合方式 — 首頁是快速開始，然後依功能分類詳細說明
- **快速開始章節**：包含外掛安裝後的首次設定步驟 + 3-5 個最常用功能快速連結
- **功能分類呈現**：分頁式細節頁面 — 主頁只有簡介和連結，點擊進入該功能的完整說明頁
- **導航方式**：左側邊欄目錄 + 右側內容（類似 WordPress Codex 樣式）

### FAQ 與疑難排解
- **FAQ 呈現方式**：混合方式 — 有獨立 FAQ 頁，但各功能頁也有「相關 FAQ」連結
- **重點 FAQ 主題**：
  - 金流設定（如何獲取 PayUNi 商店代號、測試/正式環境切換）
  - Webhook 調試（Webhook 沒有觸發怎麼辦、如何驗證 Webhook 運作正常）
  - 訂閱續扣問題（訂閱續扣失敗怎麼辦、如何更新信用卡資訊）
  - ATM 號碼查詢（客戶問我 ATM 號碼在哪、ATM 付款期限多久）
- **疑難排解工具**：提供三種工具
  - 錯誤訊息對照表（列出常見錯誤訊息及解決方法）
  - 流程圖（例：訂單未付款 → 檢查 1 → 是/否 → 檢查 2...）
  - 檢查清單（例：Webhook 無法運作：☐ URL 是否正確、☐ 簽章是否正確、☐ SSL 憑證...）

### 整合點與提示
- **說明連結方式**：三種整合方式
  - Tooltip 滑鼠按停提示（欄位標題旁放 (i) 圖示）
  - Help 連結按鈕（頁面右上角有「?」按鈕）
  - 情境式幫助訊息（特定狀態發生時顯示）
- **情境式幫助觸發時機**：
  - 第一次進入頁面（例：第一次開啟 Dashboard 時顯示「歡迎使用 PayUNi，點擊此處查看快速開始指南」）
  - 新功能上線時（新版本新增的功能有高亮提示）
- **Tooltip 詳細程度**：簡短描述（1-2 句）— 例：「交易狀態：顯示 PayUNi 交易的支付狀態」

### 說明深度與呈現
- **說明深度**：混合方式 — 基礎功能只有概念說明，複雜功能（如 Webhook 設定）有步驟教學
- **視覺輔助元素**：
  - 截圖（各功能頁面的截圖，標註重點區域）
  - 圖示說明（使用圖示解釋概念，例：訂閱續扣流程圖）
- **截圖處理**：模糊敏感資訊 — 標註截圖，但模糊商店代號、金鑰等敏感資訊

### Claude's Discretion
- 確切的頁面版型設計（只要符合左側邊欄 + 右側內容的結構）
- 快速開始的具體連結數量（3-5 個範圍內自行選擇最重要的功能）
- 圖示和圖表的具體設計風格
- FAQ 的確切排序和分組方式

</decisions>
